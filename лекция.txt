Абстракция - это выделение существенных свойств объекта, отличающих его от других объектов.
Разные цели - разные модели

Программа - множество объектов, каждый из которых обладает своими свойствами и поведением, но его внутреннее устройство скрыто от других объектов
	Нужно “разделить” задачу на объекты!
В ООП декомпозиция осуществляется по объектам

Цель занятия - изучение основных понятий ООП

Класс  - специальный тип данных, имеющий поля, методы и свойства.
	Переменная типа “Класс” называется экземпляром класса или объектом

Выполняемый файл (.exe) состоит из 3 сегментов:
-	Code segment
-	Data segment
-	Stack segment
Размеры фиксированные, после загрузки в RAM не меняются
Как правило, размер сегмента - 65 Кбайт

Определить объект - выделить ему место в “куче”, где он будет жить
Удалить объект - удалить адрес объекта.

Метод - подпрограмма объявленная внутри классового типа и определяющие его поведение


Пример:
Класс - люди
Экземпляр класса - человек
Методы - кивать, хлопать и т.д.


Инкапсуляция - единство полей, их свойств и методов
	Она позволяет объединить и защитить данные и код в объект и скрыть реализацию объекта от пользователя.

	Наследование
Для использования механизма наследования необходимо в объявлении класса class указать класс предок (базовый класс)
	class Employee : Person
{
    	}  // Корневой класс - Object;

	

Наследование -  механизм ООП при котором, класс потомок получает от класса предка все его поля, методы и свойства
Потомок не в праве удалить/изменить поля/методы

Наследование позволяет широкий класс библиотек с проверенным программным кодом и требуемой функциональностью

Класс Object является корневым, у него нет полей, только два метода - создать и уничтожить



Полиморфизм________________________________________

Полиморфизм - это механизм ООП, позволяющий одноименным методам объектов (предок и потомок) работать по разным алгоритмам.
-	Перекрытие
-	Виртуализация 
 men
Перекрытие - изменения у потомка, алгоритма одноименного метода полученного от предка

Виртуализация - изменение у предка, алгоритма одноименного метода полученного от потомка
 martin
Важнейшие механизмы ООП:
- Инкапсуляция
- Наследование
- Полиморфизм
- Абстракция


Выводы________________________________________

Плюсы:
-	основная программа - простая и понятная
-	классы могут разрабатывать разные программисты независимо друг от друга
-	повторное использование классов

Минусы:
-	Неэффективно для небольших задач
